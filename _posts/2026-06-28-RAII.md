## RAII(Resource Acquisition is Initialization)
>객체의 수명에 Resource의 수명을 bind!

|시점|동작|
|---|---|
|객체 **생성** (초기화)|리소스 **획득**|
|객체 **소멸**|리소스 **해제** (자동)|
#### 핵심 개념
Resource를 직접 열고 닫지 않고, **객체의 생성자/소멸자**에 위임한다.
Scope가 끝나거나 예외 발생해도 소멸자가 **반드시 호출**되므로 리소스 누수가 없다!


~~~ cpp
{ // RAII : Scope 생성 주기 확인!
FileHandle f("RAIIData.txt"); //acquisition, 획득
if(error) return;  // return해도 괜찮음
} // 소멸자 시행 ->리소스 자동 해제
~~~

SmartPointer에서와 같이, Class(FileHandle)안에서 RAII를 처리 한다.(아래 구현 예시 참고)

### 구현 예시

~~~ cpp
class FileHandle{
	FILE* fp;
	
public:
	FileHandle(const char* path){
		fp = fopen(path, "r"); //생성자 - 리소스 획득
	}
	~FileHandle(){
		fclose(fp);  //소멸자 - 리소스 해제(자동)
	}
	
};
~~~

## RAII 패턴 구현체들

|구현체|획득|해제 트리거|
|---|---|---|
|`FileHandle`|생성자에서 파일 오픈|스코프 종료|
|`unique_ptr`|생성자에서 메모리 할당|스코프 종료|
|`shared_ptr`|생성자에서 메모리 할당|참조 카운트 = 0|
|`lock_guard`|생성자에서 Mutex 락|스코프 종료|

> SmartPointer, Lock(Mutex) 모두 RAII 패턴의 구현체


#### 참고

- SmartPointer, Lock(Mutex), 모두 RAII패턴의 구현체
- **Resouce**는 메모리에 올라가는 모든 정보
