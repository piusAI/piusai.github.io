## extern
> 어딘가 반드시 .cpp 단 한곳에서는 반드시 정의한다.

| extern 사용한 전역변수 | extern 미사용 전역변수    |
| --------------- | ------------------ |
| 단 한 곳에서 **정의**  | 여러곳에서 **중복 정의** 이슈 |

#### 핵심 개념
일반 전역변수를 여러곳에서 정의하지 못하도록 제어해주는 keyword

~~~ cpp
// pius.h
extern int g_Value; // 어딘가에 정의 돼있다는 선언
~~~

~~~ cpp
//pius.cpp
int g_Value = 0; //실제 정의(메모리 할당)
~~~
>반드시 같은 pius.cpp에서 꼭 정의하겠다는 아니고,
>어딘가 .cpp **하나에서 반드시 정의**되어야 한다

#### extern없는 헤더 정의

~~~ cpp
int g_Value =0; //선언과 함께 정의!!
~~~

~~~ cpp
// a.cpp
#include "pius.h" //int g_Value = 0 ; 복붙

// b.cpp
#include "pius.h" //int g_Value = 0 ; 복붙
~~~

~~~ cpp
error LNK2005: g_Value already defined in a.cpp
~~~

: include한 모든 .cpp마다 `g_Value`가 생겨 **중복 정의 링크 에러** 발생

---
#### extern 활용

~~~ cpp
//pius.h
extern int g_Value; // 선언만 (메모리 할당 X)
~~~
a.cpp과 b.cpp에 둘다 include 하더라도,
"어딘가에 있다"는 정보만 복사 -> 중복 없음!

~~~ cpp
// ~.cpp
int g_Value =0; //메모리 할당은 딱 한번 여기서만!
~~~


### 한줄 요약

> `extern` 없이 헤더에 정의 : include 수만큼 **중복 정의**
> 
> `extern`있으면 : 선언만 퍼지고, 정의는 딱 한곳

extern있으면 **정의 / 선언을 나누어서** 사용하는게 좋겠다.
그것이 extern의 존재 이유자체이니까!
