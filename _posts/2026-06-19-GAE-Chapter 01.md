#### 04 툴-엔진 구조아키텍쳐

**[1] 언리얼 엔진구조**

언리얼 엔진 : 툴과 런타임이 같은 엔진, 같은 자료구조

- Editor, Game Runtime이 동일한 엔진코드, 데이터 구조 사용  
- Actor, World, Component, Render/Physics pipeline 동일

**장점** : WYSIWYG 구조로 즉시 반영, 디버깅 쉬움, 코드/데이터 일관성 최고  
**단점** : 엔진이 무겁다, 빌드/에디터 크기가 크다

  

**[2] 프레임 워크 공유형**

유니티, 고도엔진 : 툴과 게임이 같은 기반의 프레임 워크를 사용

- 에디터, 런타임은 분리된 프로그램

- 공통 엔진 코어(API, 시스템)만 공유

**장점** : 툴제작 쉬움, 구조 단순  
**단점** : 에디터와 실제 게임 차이 발생, WYSIWYG 정확도 떨어짐, 디버깅 분리

  

**[3] 독립 런타임 + 전용툴**

Quake 엔진 : "툴과 엔진 완전 분리"

-툴은 데이터 생성기, 런타임 엔진은 실행 전용  
**장점** : 런타임 최적화 최고, 툴 가볍다  
**단점** : 반복 작업 느림, 디버깅 불편, 툴-엔진 불일치 가능성
