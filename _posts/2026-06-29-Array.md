배열을 활용할때 좀 헷갈리는 부분들이 있다.
특히 **어디에 메모리가 잡히는가**, **어떻게 해제**하는가에 대해서 한번씩 헷갈렸다.


### 구조체, Structure에서의 활용 - 고정배열


``` cpp
struct Userstate{
	int hp;
	int happiness;
};

Userstate _userstate; // 구조체 1개 생성
Userstate arr[6]; //구조체 6개 연속 생성
// Memory : [ ] [ ] [ ] [ ] [ ] [ ]
```

arr[6]은 UserState 크기 만큼 공간이 6개 연속 잡힌다.
포인터와 배열이 비슷하게 동작하는 이유가 여기서 나온다
`arr[i]`는 결국 `*(arr + i)` : `arr크기* i` offset(이동)와 같다

그렇기 때문에 `ptr[i]`가 가능한것

--- 
#### 배열 선언 3가지 방식 비교

|          | 고정 배열               | 포인터 배열               | 동적 배열                                |
| -------- | ------------------- | -------------------- | ------------------------------------ |
| 선언       | `UserState arr[10]` | `UserState* arr[10]` | `UserState* arr = new UserState[10]` |
| 메모리      | Stack               | Stack(지역) / Data(전역) | Heap                                 |
| 범위기반 for | ✅                   | ✅                    | ❌ / 포인터의 길이 정보를 모르기때문                |
| delete   | 불필요                 | `delete` 10번         | `delete[] arr` 1번                    |


---
### 포인터 배열 - new / delete 각각 n번

``` cpp
#include <iostream>
using namespace std;


struct UserState
{
	int hp;
	int happiness;
};


UserState* arr[10]; // Data(BSS) memory 영역
int main()
{
	srand(static_cast<unsigned int>(time(0)));
	for (int i = 0; i < 10; i++)
	{
		arr[i] = new UserState;  //하나씩 new 했으니

		arr[i]->hp = 20 * (rand() % 4 + 1);
		arr[i]->happiness = 200 * (rand() % 5);
	}

	for (UserState* US : arr)  //범위기반 for 가능
	{
		cout << "hp : " << US->happiness << endl;
	}
	
	for(int i = 0 ;  i< 10; i ++)
	delete arr[i]; //하나씩 삭제해야함

}
```

`new`를 10번 했다면 `delete`도 10번 해야한다!
각 개체가 heap의 뿔뿔이 흩어진 위치에 따로 잡혀있기 때문이다.

```
// Heap memory(연속성 보장 x)

힙 메모리 (단편화됨)
[UserState] ... [UserState] ... [UserState] ...
     ↑               ↑               ↑
   arr[0]          arr[3]          arr[7]   (연속 보장 없음)
```

> new, delete는 **항상 1:1**
 10번 `new` 했는데, `delete` 한번만 하게된다면 메모리 누수(memory leak) 발생



### 동적 배열 - new / delete 한 번씩

###### new-delete 한번만에 - 동적 배열

``` cpp
#include <iostream>
using namespace std;


struct UserState
{
	int hp;
	int happiness;
};


int main()
{
	UserState* arr= new UserState[10];
	srand(static_cast<unsigned int>(time(0)));
	for (int i = 0; i < 10; i++)
	{
		// 포인터와 배열은 비슷 : [i]와 같이 활용시 offset으로 옆칸이동
		// ->가 아닌 .으로 구조체 접근
		arr[i].hp = 20 * (rand() % 4 + 1);
		arr[i].happiness = 200 * (rand() % 5);
	}

	for (int i = 0 ; i < 10 ; i ++)
	cout << "hp : " << arr[i].happiness << endl;
		
	delete[] arr; //

}
```
한번에 `UserState* arr = new UserState[10]`으로 할당했다면
Heap에 *연속된 공간*을 한번에 잡는다.


```
// 힙 메모리 (연속 보장)
[UserState][UserState][UserState]...[UserState]
    arr[0]    arr[1]    arr[2]         arr[9]
       ↑
      arr (시작 주소만 들고 있음)

```
-> `sizeof(UserState) * 10`만큼한번에 Heap에서 연속 공간을 확보!
`delete[] arr;`로 delete 한번에 끝낸다
`delete arr`로 쓰면 첫번째 객체만 소멸되고 나머지는 누수되는점 주의!
반드시 `[]`를 붙혀야한다!

> 연속된 메모리 배열
사실 조금 더 정확히 말하자면 `delete arr`는 Undefined Behavior이다.  
배열을 사용했다면 반드시 `delete[]`로 해제 해야한다!



### 한눈 비교

|방식|힙 레이아웃|delete|
|---|---|---|
|`UserState* arr[10]` + 개별 `new`|뿔뿔이 흩어짐|`delete arr[i]` × 10|
|`UserState* arr = new UserState[10]`|연속|`delete[] arr` × 1|
- 객체 수가 고정이고 함수 스코프 내에서만 쓴다면 → **고정 배열** (`UserState arr[10]`)
- 런타임에 크기가 결정되거나 함수 밖에서도 유지해야 한다면 → **동적 배열** (`new[]` +`delete[]`)

- **포인터 배열** 의 예시
-   - 서로 다른 객체를 가리켜야할때
-   - 객체를 개별 생성/삭제 해야할때
-   - 다형성 사용시


>보통 `new` / `delete`를 직접 쓰는 대신  
RAII래퍼인, `std::vector`, `std::unique_ptr` 같은 것을 쓰는것이 일반적!
그렇다해도 포인터와 메모리 구조의 base이해가 필요하기에 원리 중요!
