### Stack
- 언제 사라질지 모르는 불안전한 공간
- 가볍게 임시로 활용하기에 좋은 메모리 영역
- EX) 지역변수

RSP : Register Stack Pointer

-> 호출했다가 종료되는 시점에, 다시 돌려준다!
고정 주소가 생길 수 없다.

```
main( func1( func2() ) )
```


<table width="100%" style="table-layout: fixed; border-collapse: collapse; border: none;"> <tr style="border: none;"> <td width="50%" style="text-align: center; border: none; padding: 5px;"> <img src="/assets/postimg/Memory/Callstack.jpg" alt="VS003" style="width: 100%; max-width: 100%; height: auto;"> <br><strong>CallStack</strong> </td>  </tr> </table>
- 01 함수 호출 : StackFrame 쌓임(PUSH) : 매개변수 / 로컬 / 주소 새프레임에 쌓임
- 02 함수 끝 : StackFrame 사라짐 (POP) : RSP복구해서 프레임 전체 사라짐
- 03 복귀 : 호출한 이전 함수로 돌아간다

<table width="100%" style="table-layout: fixed; border-collapse: collapse; border: none;"> <tr style="border: none;"> <td width="50%" style="text-align: center; border: none; padding: 5px;"> <img src="/assets/postimg/Memory/stackclean.png" alt="VS003" style="width: 100%; max-width: 100%; height: auto;"> <br><strong>32Bit Stack</strong> </td>  </tr> </table>
```
esp, 8 //스택 정리 (인자 2개 x 4바이트 = 8)
```

하지만..
64bit 공간으로 간다면, 스택 접근 비용이 2배로 들어가기에,

포인터가 64bit -> Push/Pop으로 Stack에 8byte씩 넣어야함.
-> 32bit때는 4byte씩 Push로 했지만, 64bit에서는 8Byte씩이라 스택 접근 비용이 2배
-> "Stack에 올리지말고 CPU에 붙어있는 레지스터에 넣자"

일시적으로 쓰는 것이기 때문에 Stack은 크게 아껴봤자 필요없다!

***Stack Oberflow?***
잘못 된 주소로 복귀 / 접근 위반
Stack Memory 범위 초과해 접근시 발생
-> 지역변수, 리턴주소 손상되어 잘못된 주소 복귀, 접근 위반!
